# 🚀 Варианты развёртывания

Выбери, где и как развертать Goh Hosting.

---

## 1️⃣ Production на VPS

**Сервер:** 2.26.54.227 (Ubuntu/Debian)  
**Функционал:** ✅ Полный (Docker, управление ботами, автообновление)  
**Стоимость:** Платный хостинг  
**Сложность:** Средняя (требует SSH и Docker)

### Минусы
- Нужно обслуживать сервер
- Платный хостинг
- Требует настройки DNS/ngrok

### Плюсы
- Полный контроль
- Все функции работают
- Высокая надёжность

### Запуск
```bash
ssh root@2.26.54.227
cd /root/goh-hosting
docker compose up -d
```

## 2️⃣ Разработка на Daytona

**Платформа:** Daytona IDE (облачная)  
**Функционал:** ✅ Веб-панель + бот (❌ управление Docker)  
**Стоимость:** Бесплатно (первые часы), затем платно  
**Сложность:** Очень низкая (5 минут)

### Плюсы
- Быстрый старт (5 минут)
- Встроенная IDE и Terminal
- Публичный URL для демо
- Git интеграция встроена
- Бесплатно первое время

### Минусы
- Docker управление недоступно
- Не подходит для production
- Может быть остановлен после неактивности
- Ограниченные ресурсы

### Запуск
```bash
# Через UI: https://app.daytona.io
# New Workspace → GitHub: gohhost.git

# Или через SSH:
bash daytona-deploy.sh
```

## 3️⃣ Локально (Windows/Mac/Linux)

**Платформа:** Локальная машина  
**Функционал:** ✅ Полный (если установлен Docker)  
**Стоимость:** Бесплатно  
**Сложность:** Средняя

### Плюсы
- Полная разработка локально
- Бесплатно
- Быстро тестировать изменения

### Минусы
- Нет публичного доступа (нужен ngrok)
- Требует Docker и Node.js
- Боты недоступны для пользователей

### Запуск
```bash
# 1. Установить Docker Desktop
# 2. Установить Node.js 22+
# 3. Клонировать репо
git clone https://github.com/HouseGram-code/gohhost.git
cd gohhost

# 4. Настроить .env
cp .env.example .env
# вставить: ngrok token, bot token, etc

# 5. Запустить
docker compose up -d
npm run dev
```

## 4️⃣ GitHub Actions (CI/CD деплой)

**Платформа:** GitHub Actions → VPS  
**Функционал:** ✅ Автоматический деплой при push  
**Стоимость:** Бесплатно (GitHub + платный хостинг)  
**Сложность:** Средняя (требует Secrets)

### Плюсы
- Автоматический деплой
- Не нужно ручных команд
- Логи деплоя видны в GitHub

### Минусы
- Требует настройки SSH-ключей
- Требует платный хостинг (VPS)

### Запуск
```bash
# 1. Добавить SSH-ключи в Secrets GitHub
# Settings → Secrets → New repository secret

# Секреты:
SERVER_HOST = 2.26.54.227
SERVER_USER = root
SERVER_SSH_KEY = (приватный SSH-ключ)

# 2. Workflow готов в .github/workflows/deploy.yml
# 3. При каждом push в main — автоматический деплой
```

📖 [.github/workflows/deploy.yml](./.github/workflows/deploy.yml)

## 5️⃣ Heroku / Railway / Fly.io

**Платформа:** PaaS (Platform as a Service)  
**Функционал:** ⚠️ Частичный (веб + бот, ❌ Docker)  
**Стоимость:** Платно ($5-50/месяц)  
**Сложность:** Низкая

### Плюсы
- Очень легко развернуть
- Встроенный SSL
- Автоматический скейлинг
- CI/CD встроен

### Минусы
- Docker управление недоступно
- Дороговато
- Ограничения на ресурсы

### Запуск (Railway пример)
```bash
# 1. Создать проект на railway.app
# 2. Conectить GitHub репо
# 3. Добавить переменные окружения
# 4. Deploy
```
